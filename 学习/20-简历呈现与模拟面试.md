## 三分钟项目介绍（背诵模板）

> 这是一个基于 **STM32F407 + FreeRTOS** 的双机智能门锁项目。  
> **主机**负责 LVGL 触摸屏 UI、W25Q 用户存储、NFC/指纹开锁、ESP8266 连接阿里云 MQTT，并通过 **RS485** 把用户和指纹模板同步到**从机**侧门。  
> 从机无 WiFi，开锁后通过自定义 RS485 协议通知主机上报云端，手机 App 经 Node 后端与门锁互通。  
> 我负责/熟悉了××模块（填你真实做过的），例如 RS485 协议调试、ui_v3 界面、或 NFC bring-up。

## 简历示例（ bullets ）

- 基于 **STM32F407 + FreeRTOS** 双机智能门锁，主机集成 **LVGL ui_v3**、**ESP8266 阿里云 MQTT**、**RS485 主从同步**
- 设计 **RS485 自定义协议**（CRC16-MODBUS、指纹 512B 分片镜像、从机开锁上报）
- 实现 **指纹代匹配架构**：从机采图传特征、主机 Search，保证分布式场景用户数据**单一权威源**

- 独立 **硬件自检工程**（W25Q/NFC/触屏/账号诊断），缩短焊板 bring-up 周期

## 模拟面试题与参考答案

### 基础理解

**Q1：为什么用 RS485 不用普通 UART 直连？**  
A：RS485 差分传输，抗干扰、可长线；半双工两线组网方便主从。普通 TTL UART 适合板内短距离。

**Q2：Host 和 Client 怎么区分编译？**  
A：双工程 `Host/`、`Client/`，或同一工程改 `APP_RS485_NODE_ROLE` 宏为 MASTER/SLAVE。

**Q3：用户数据存在哪里？掉电怎么办？**  
A：W25Q16 外部 Flash，魔数 USR1；修改异步刷盘；启动 `users_storage_load`。

---

### 协议与同步

**Q4：RS485 帧格式和 CRC 怎么算？**  
A：A5 5A + dst + src + seq + cmd + len + payload + CRC16-MODBUS，覆盖 dst 到 payload 末字节，初值 FFFF，多项式 A001。

**Q5：指纹模板多大？怎么同步到从机？**  
A：512B，分 4 片每片 128B，命令 FP_TEMPLATE，片间有 gap 等 AS608 写完。

**Q6：从机断电后怎么恢复数据？**  
A：上电发 MIRROR_SYNC_REQ，主机全量下发用户和指纹模板。

**Q7：代匹配流程时序？超时怎么办？**  
A：从机 Img2Tz → FP_MATCH_CHAR 分片 → 主机 Search → 应答；超时 APP_FP_SLAVE_MATCH_RS485_MS，可 fallback。

---

### RTOS 与架构

**Q8：为什么开锁弹窗要单独 pump？**  
A：LVGL 非线程安全，只能在 GuiTask 操作 UI。

**Q9：任务优先级怎么定的？**  
A：GUI 最高保证流畅；Cloud 中等；Storage 最低避免擦 Flash 卡 UI。

**Q10：CloudTask 栈为什么最大？**  
A：AT + WiFi 扫描 + MQTT 嵌套深，默认 4096 字。

---

### 驱动与硬件

**Q11：软 SPI 和硬件 SPI 各用在哪？**  
A：硬件 SPI1 给 ILI9341；W25Q 和 MFRC522 用软 SPI 另一组 GPIO。

**Q12：W25Q 为什么启动时要早读？**  
A：PA15 与 JTAG 复用；避免与 LCD 初始化争用；尽早有用户数据。

**Q13：MFRC522 为什么要软复位？**  
A：上电时序要求，否则寻卡失败但版本寄存器可能正常。

---

### 可靠性与调试

**Q14：云上报失败怎么办？**  
A：RAM 队列 + app_unlock_flash_queue Flash 备份，重连后重试。

**Q15：怎么区分触屏坏了还是 UI 问题？**  
A：test/test 看 [TP]；PASS 则查双击逻辑或 ui_v3 手势。

**Q16：从机死机怎么办？**  
A：IWDG 复位；重新 mirror 同步。

---

### 扩展思考（加分项）

**Q17：如何防止 RS485 重放攻击？**  
A：现状无加密/序号防重放；可加时间戳+HMAC 或递增 nonce。

**Q18：NFC UID 克隆风险？**  
A：本地白名单，不防克隆；可加强 M1 扇区密钥认证。

**Q19：用户从 64 扩到 1000 怎么改？**  
A：Flash 分区扩容、索引结构、可能改 AS608 策略或特征比对放主机。

**Q20：ui_v3 为什么不用纯 Guider？**  
A：复杂手势/动效/导航栈难维护；手写模块化更可控，可与浏览器原型对齐迭代。

## 结业自测（终极 5 题）

1. 用一句话向非技术人员介绍这个项目。
2. 你最熟悉的文件是哪个？为什么？
3. 如果 RS485 A/B 线接反了会怎样？
4. 主机录入新指纹后从机何时能刷该手指开锁？
5. 你认为项目最大的技术债务是什么？

### 参考答案

1. 「双门智能锁：大门能连手机管理，侧门也能刷卡，整套嵌入式联网系统。」
2. 开放题；建议能讲 `app_rs485_proto.c` 或 `main.c` 任务表。
3. 可能完全无应答或 CRC 错；部分收发器反接仍通信但抗干扰差。
4. 主机 Store 成功 → FP_TEMPLATE 4 片写完从机 AS608 → mirror commit 完成。
5. 开放题；如 cloud_aliyun_at.c 过大、密钥硬编码、RS485 无加密等。
