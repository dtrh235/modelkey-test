# 第 5 课：电容触摸 FT6336

> **阶段**：存储与显示 | **建议课时**：3～4 小时

---

## 上节复习

- LCD + `lv_port_disp` 显示链路
- SPI1 专供屏幕，与 W25Q 软 SPI 分开

**预习下节（第 6 课）**：触摸能点后，NFC 是另一种「输入」——刷卡代替按键。

---

## 学习目标

- 理解软 I2C 读 FT6336 坐标
- 知道 `lv_port_indev` 如何把触摸交给 LVGL
- 区分硬件问题与「双击进开锁」等软件逻辑

---

## 本课要读的文件

| 文件 | 关注什么 |
|------|----------|
| `Host/Drivers/BSP/TOUCH/ctiic.c` | 软件 I2C 时序 |
| `Host/Drivers/BSP/TOUCH/ft5206.c` | FT6336 驱动（文件名历史遗留） |
| `Host/Middlewares/LVGL/.../lv_port_indev.c` | 输入设备注册 |
| `Host/User/app_touch_ui.c` | 首页双击等业务逻辑 |
| `Host/User/app_config.h` | `APP_TP_MIRROR_X`、`APP_TP_FT6336_CAP_ONLY` |

---

## 专有名词

| 名词 | 解释 |
|------|------|
| **I2C** | 两线 SCL/SDA，可多设备挂在总线上 |
| **软 I2C** | GPIO 模拟起始/停止/ACK，`ctiic.c` |
| **FT6336** | 电容触摸芯片，芯片 ID 常见 `0x51` |
| **INT (PB1)** | 触摸中断，有触摸时引脚变化 |
| **坐标镜像** | `APP_TP_MIRROR_X`：竖屏时 X 是否 `width - x` |

---

## 功能怎么实现

```
ft5206 读 ID / 读坐标
    → touch.c 抽象层
    → lv_port_indev read_cb（周期性采样）
    → LVGL 命中测试、按钮事件
    → app_touch_ui / ui_v3 手势（业务层）
```

### 引脚

- I2C：PB6=SCL, PB7=SDA
- RST：PE8
- INT：PB1

### 业务层：首页双击

`app_touch_ui.c` 设计：首页「开锁」按钮需**点两次**（第一次选中，第二次进入），防止误触。  
因此 `[TP] PASS` 但「进不了开锁」不一定是硬件坏。

---

## 动手实验

1. 第 2 课触屏自检，确认 `id=0x51` 和 `DOWN/UP` 坐标。
2. 主程序中试单击 vs 双击首页开锁按钮。
3. 若坐标左右反了，改 `APP_TP_MIRROR_X` 重新编译。

---

## 易错点 / 困难点

- I2C 无 ACK：查 3.3V、PB6/PB7、PE8 复位时序。
- 坐标偏移/镜像：调 `APP_TP_MIRROR_X`，不同 PCB 贴法不同。
- ui_v3 还有 `app_ui_v3_gesture.c` 滑动手势，与旧 `app_touch_ui` 可能并存。
- `APP_TP_FT6336_CAP_ONLY=1` 裁掉电阻屏驱动以省 Flash。

---

## 本课总结

- FT6336 走软 I2C PB6/PB7
- LVGL 输入在 `lv_port_indev`
- 业务触摸逻辑在 `app_touch_ui` / ui_v3 手势
- 自检 `[TP]` 与主程序 UI 行为要分开判断

---

## 本课提问

1. FT6336 正常时芯片 ID 是多少？
2. `APP_TP_MIRROR_X` 解决什么问题？
3. 首页进开锁为什么要双击？
4. `lv_port_indev.c` 和 `ft5206.c` 谁依赖谁？

---

## 参考答案

1. `0x51`（自检日志 `id=0x51`）。
2. 不同 PCB/屏安装方向导致触摸 X 坐标左右颠倒。
3. `app_touch_ui` 防误触设计：第一次选中、第二次确认进入开锁流程。
4. `lv_port_indev` 调用触摸层/`ft5206` 读坐标，再交给 LVGL 处理点击。
