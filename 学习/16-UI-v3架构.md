## 屏幕枚举（`ui3_scr_id_t`）

| ID | 屏幕 | 作用 |
|----|------|------|
| UI3_SCR_HOME | 首页 | 时钟、菜单、开锁入口 |
| UI3_SCR_UNLOCK | 开锁 | 账号密码 |
| UI3_SCR_ADMIN | 管理员 | 管理员验证 |
| UI3_SCR_MENU | 菜单 | 功能列表 |
| UI3_SCR_ADD_USER 等 | 用户管理 | 增删改查 |
| UI3_SCR_WIFI | WiFi | 配网 |
| UI3_SCR_PAIR | 配对 | App 六位数 |
| UI3_SCR_NFC_MGMT | NFC 管理 | 录入/解绑 |
| UI3_SCR_FP_MGMT | 指纹管理 | 录入/解绑 |

## 模块职责

```
app_ui_v3.c        总入口，与 main GuiTask 对接
app_ui_v3_screens  每屏 layout + 事件
app_ui_v3_widgets  卡片、列表行、WiFi 条等复用组件
app_ui_v3_gesture  滑动返回等
app_ui_v3_users    录入状态机，对接 NFC/指纹
app_ui_v3_theme    纸质风配色
app_ui_v3_anim     过渡动画
```

### 导航栈

`ui3_state_t.stack[UI3_NAV_STACK_MAX]`（最大 8 层）：  
进入子页 push，返回 pop，避免「返回键不知道回哪」。

### 与旧 UI 共存

`APP_UI_V3_ENABLE=1` 时走 ui_v3；`=0` 时走 Guider `setup_ui`。  
部分录入仍关联 `g_app_scr`、`screen8` 等旧状态，`main.c` 中 `app_enroll_flow_active()` 同时判断 ui_v3 与旧屏。

## GuiTask 中的 ui_v3

```c
app_ui_v3_tick();
lv_timer_handler();
app_ui_v3_poll();
```

- `tick`：时钟、动效定时
- `poll`：业务轮询、状态刷新
- 周期 5ms

## 动手实验

1. 浏览器打开 `lvgl/index.html`，对照 HOME/UNLOCK 屏。
2. 在 `app_ui_v3_widgets.c` 从你现在关注的行（如列表行组件）向前后各读 50 行。
3. 切换 `APP_UI_V3_ENABLE` 0/1 看编译差异（勿频繁切换，选一条主线深读）。

---

## 易错点 / 困难点

- `widgets.c` 1000+ 行，**按屏/按组件**读，不要一次通读。
- 手势 + 物理键 `app_key_ui.c` 可能双重处理事件。
- 所有 LVGL 操作必须在 GuiTask（复习第 9、10 课）。
- 改样式要看 `theme.c`，避免在每个 widget 里硬编码颜色。

---

## 本课总结

- ui_v3 = screens + widgets + theme + anim + gesture
- 导航栈管理页面流
- 先浏览器原型再看 C 实现，事半功倍

## 参考答案

1. 8。
2. 宏 `APP_UI_V3_ENABLE`：1 用 ui_v3，0 用 Guider `setup_ui`。
3. `tick` 偏时钟/动效定时；`poll` 偏业务轮询与界面刷新（均在 GuiTask）。
4. `Host/guider/tools/gen_ui_v3_fonts.ps1` + symbols txt 裁剪生成 `lv_font_ui_v3_*.c`。
