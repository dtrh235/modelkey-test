# modelkey 项目目录结构

> STM32F407 智能门锁双机工程。本文只列目录骨架，模块职责与数据流后续再细化。

## 仓库顶层

```
modelkey/
├── Host/              # 主控固件（RS485 Master）
├── Client/            # 从机固件（RS485 Slave）
├── test/              # 独立测试工程
├── lvgl/              # LVGL 参考副本（主工程内已自带）
├── JoyfulZone/        # 手机 App 原型 + Node 后端（非固件）
├── 学习/              # 本目录：学习笔记与结构说明
├── README.md
└── CHANGELOG.md
```

## Keil 工程入口

| 固件 | 工程文件 |
|------|----------|
| 主控 Host | `Host/Projects/MDK-ARM/stm32f407_lvglport.uvprojx` |
| 从机 Client | `Client/Projects/MDK-ARM/stm32f407_lvglport.uvprojx` |
| NFC/RS485 测试 | `Host/test/nfc_enroll_rs485/MDK-ARM/nfc_enroll_test.uvprojx` |

## Host / Client 通用骨架

`Host/` 与 `Client/` 目录结构相同，主从由 `User/app_config.h` 中 `APP_RS485_NODE_ROLE` 区分。

```
Host/  （Client/ 同理）
├── Projects/
│   └── MDK-ARM/                  # Keil 工程 (.uvprojx / .uvoptx)
├── Output/                       # 编译产物 (.hex / .axf / .map)
├── Drivers/
│   ├── BSP/                      # 板级外设驱动
│   ├── CMSIS/                    # ARM CMSIS
│   ├── STM32F4xx_HAL_Driver/     # ST HAL 库
│   └── SYSTEM/                   # delay / sys / usart
├── Middlewares/
│   ├── FreeRTOS-Kernel/          # RTOS
│   └── LVGL/
│       └── LVGL_SRC/lvgl/        # LVGL 源码 + examples/porting 移植层
├── guider/                       # NXP GUI Guider 生成界面
│   ├── generated/                # 自动生成的 screen、字体、图片、事件
│   ├── custom/                   # 自定义扩展
│   └── tools/                    # 字体/资源生成脚本
├── test/                         # 板级单项测试（见下文）
└── User/                         # 应用层业务代码
    └── ui_v3/                    # 新 UI（仅 Host 当前使用）
```

## 主从关系（一句话）

| | Host | Client |
|---|------|--------|
| RS485 角色 | Master (0x01) | Slave (0x02) |
| WiFi / 云 | 有 | 无 |
| UI | Guider 全屏 + ui_v3 | 精简（主要 screen_1） |
| 开锁上报 | 直接 MQTT | 经 RS485 通知 Host |
