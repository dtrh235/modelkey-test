## 通俗讲解

主程序像一辆整车，自检工程像**单项体检**：

- 只测 W25Q → 排除 Flash 接线问题
- 只测触屏 → 排除 I2C/FT6336 问题
- 测账号 `1/1111` → 区分「硬件坏了」还是「Flash 密码被改了」

**重要**：自检默认用 **RS485 口 115200** 看日志（PC6/PC7），不是 PA9 调试口。

---

## 功能怎么实现

`test_main.c` 根据宏依次调用：

| 宏 | 测试内容 |
|----|----------|
| `TEST_ENABLE_W25Q_HW` | JEDEC ID + 擦写读 |
| `TEST_ENABLE_TOUCH_HW` | FT6336 ID + 触摸坐标 |
| `TEST_ENABLE_NFC_HW` | MFRC522 读卡 UID |
| `TEST_ENABLE_UNLOCK_SW` | 读 Flash 用户表，模拟 `1/1111` 校验 |

---

## 接线表（自检工程）

| 功能 | 引脚 |
|------|------|
| RS485 日志 | PC6→DI, PC7←RO, PC8=DE |
| W25Q16 | **PE0=CS**, PB3=SCK, PB4=MISO, PB5=MOSI |
| MFRC522 | PB12～15 + PD1=RST |
| FT6336 | PB6/PB7 I2C, PB1=INT, PE8=RST |

## 动手实验

### 实验 A：只测 W25Q

`test/test/User/app_config.h`：

```c
#define TEST_ENABLE_W25Q_HW    1
#define TEST_ENABLE_NFC_HW     0
#define TEST_ENABLE_TOUCH_HW   0
```

烧录后 RS485 115200，成功应见：

```
[W25Q] STEP2 PASS: chip ID EF4015 (W25Q16)
[W25Q] PASS erase/write/read
```

### 实验 B：只测触屏

```c
#define TEST_ENABLE_W25Q_HW    0
#define TEST_ENABLE_TOUCH_HW   1
```

手指点屏，应见 `[TP] DOWN x=.. y=..`。

## 日志判定速查

### 触屏 `[TP]`

| 日志 | 含义 |
|------|------|
| `PASS hardware: FT6336 id=0x51` | I2C + 芯片正常 |
| `FAIL hardware: I2C no ACK` | 触屏硬件/接线问题 |
| `[TP] DOWN` 有，主程序仍无效 | **软件**：Host 首页须**双击**开锁 |

### 开锁 `[AUTH]`

| 日志 | 含义 |
|------|------|
| `simulate unlock => PASS` | Flash 数据正确 |
| `=> 主程序进不了页：优先查触屏/双击` | 数据 OK，问题在触屏或 UI |
| `simulate unlock => FAIL` | 密码不是 1111 或用户表异常 |

---

## 易错点 / 困难点

| 现象 | 常见原因 |
|------|----------|
| `[TP] FAIL: I2C no ACK` | 触屏线、3.3V、RST(PE8) |
| W25Q 测试用 **PE0=CS**，主工程用 **PA15=CS** | 焊板版本不同，别混用接线 |
| 有 `[TP] PASS` 但点不了按钮 | 双击逻辑或 ui_v3 手势，不是硬件 |

---

## 本课总结

- 自检工程 = 硬件/数据**分层**排查
- 用宏开关只测一项，降低干扰
- `[TP]` / `[AUTH]` 日志是排障字典
- 主程序进不了页：先跑自检区分硬件 vs 数据 vs UI

---

## 本课提问

1. 只测 W25Q 时三个 `TEST_ENABLE_*` 宏应怎么设？
2. `[AUTH] simulate unlock => PASS` 但主程序仍进不了开锁页，优先查什么？
3. 测试工程 W25Q 的 CS 脚和主工程一样吗？
4. `TEST_W25Q_CONT_READ` 有什么用？

---

## 参考答案

1. `TEST_ENABLE_W25Q_HW=1`，`TEST_ENABLE_NFC_HW=0`，`TEST_ENABLE_TOUCH_HW=0`。
2. 触屏硬件或 UI 操作（Host 首页须双击开锁）；数据已证明正常。
3. 不一样：测试用 **PE0**，主工程 BSP 用 **PA15**（与 JTAG 复用，需 SW 调试）。
4. 连续读 Flash，配合示波器看 SPI 波形（SCK/MOSI/MISO/CS），用于硬件调试。
