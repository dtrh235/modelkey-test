## 功能怎么实现

### 主机本地开锁

```
鉴权通过（密码/NFC/指纹/临时密码）
    → app_unlock_event_* 入队（非 GUI 任务）
    → GuiTask: app_unlock_event_gui_pump() 弹窗
    → UART4 发字符 '1' @115200（PC10）驱动门锁
    → CloudTask: MQTT 上报 unlock_record
```

### 从机开锁

```
Client 本地鉴权通过
    → RS485 SLAVE_UNLOCK_NOTIFY（15 字节）
    → Host app_rs485_master_poll_incoming 解析
    → 同样走上报云端（device_id=2）
```

### 线程安全（重点）

**LVGL 非线程安全**：CloudTask、HomeAuth、FpTask 等**不能直接** `lv_msgbox_create`。  
必须入队，由 GuiTask 的 `app_unlock_event_gui_pump()` 统一处理 UI。

### 上报可靠性

云断线时：RAM 队列 + `app_unlock_flash_queue.c` 持久化，连上后重试。

## 动手实验

1. 在 `app_unlock_event.c` 找队列结构和 `gui_pump` 实现。
2. 密码开锁一次，若有云端环境观察后端 `unlock_records.json`。
3. 搜索 `app_unlock_uart4` 看 UART4 发送内容。

---

## 易错点 / 困难点

- 在错误任务里调 LVGL → 死机或花屏。
- 忘记从机开锁也要走 Host 才能上云（Client 无 WiFi）。
- `method_id` 与阿里云物模型 `unlock_method` 必须一致。
- 首页生物识别在 **HomeAuth** 任务，不在 GuiTask。

---

## 本课总结

- 开锁 = 鉴权 + 事件队列 + GUI 反馈 + UART4 + 云
- **线程安全**是本项目关键设计点
- 从机开锁经 RS485 汇总到主机上云

---

## 本课提问

1. 为什么开锁弹窗必须在 GuiTask？
2. UART4 在本项目里干什么？
3. `method_id` 密码/NFC/指纹/临时密码各是多少？
4. 云上报失败怎么办？

---

## 参考答案

1. LVGL 只能在注册它的任务里调用，否则死机或花屏。
2. PC10 发字符 `'1'` @115200，通知门锁执行机构开锁。
3. 1 密码、2 NFC、3 指纹、5 临时密码（见 `app_rs485_proto.h`）。
4. RAM 队列 + `app_unlock_flash_queue` 持久化，连上云后重试。
